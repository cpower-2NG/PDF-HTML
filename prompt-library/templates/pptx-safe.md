# PPTX 稳定版

```text
请基于下面的信息，生成一套优先考虑 PPTX 稳定转换的 HTML 演示文稿方案。

主题：
[主题]

受众：
[受众]

输出目标：
[只要视觉一致 / 需要可编辑 PPTX / 需要 HTML + PDF + PPTX 三种输出]

内容范围：
- [必须包含的内容 1]
- [必须包含的内容 2]
- [必须包含的内容 3]

转换要求：
- 每页只表达一个主意，内容离散，方便拆页
- 保持文本可选、可复制、可搜索
- 尽量使用线性 DOM 结构，内容顺序要和阅读顺序一致
- 避免复杂叠层、sticky、滚动容器、hover 依赖和强动画依赖
- 图表优先用 SVG 或简单矢量，不要把关键信息烤成截图
- 先保证浏览器打印预览稳定，再考虑 PPTX 转换
- 如果需要可编辑 PPTX，请明确说明推荐的转换路径和编辑边界

输出顺序：
1. 方案摘要
2. PPTX 风险点
3. 页面大纲
4. HTML 结构或完整 HTML
5. 转换建议
```
